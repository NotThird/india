# Global Tech Stack

## When to use this skill:

- When selecting libraries or packages for new features
- When ensuring code aligns with the project's framework conventions
- When working with the project's database or ORM
- When implementing frontend features using the established UI framework
- When writing tests using the project's test framework
- When configuring build tools, linters, or formatters
- When deploying or configuring CI/CD pipelines
- When integrating authentication, email, or monitoring services
- When reviewing code for technology stack consistency
- When onboarding to understand the project's technologies
- When making architectural decisions about new features
- When choosing between different implementation approaches

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global tech stack.
