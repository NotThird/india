# Frontend Components

## When to use this skill:

- When creating new UI components
- When modifying existing component files
- When working on files in `components/`, `ui/`, `widgets/`, or similar directories
- When working with React (.jsx, .tsx), Vue (.vue), Svelte (.svelte), or Angular components
- When defining component props and their types
- When managing local component state
- When deciding where state should live (local vs. lifted)
- When composing smaller components into larger ones
- When designing component APIs and interfaces
- When naming components and their props
- When documenting component usage and examples
- When refactoring monolithic components into smaller pieces
- When implementing component encapsulation
- When setting default prop values
- When reviewing component design for reusability

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend components.
