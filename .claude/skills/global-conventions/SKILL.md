# Global Conventions

## When to use this skill:

- When organizing files and directory structures
- When creating or updating README files and documentation
- When writing git commit messages
- When creating pull/merge requests with descriptions
- When managing environment variables and configuration
- When adding, updating, or removing dependencies
- When setting up or following code review processes
- When implementing feature flags for incomplete features
- When maintaining changelogs or release notes
- When defining testing requirements for merging
- When ensuring secrets and API keys are not committed
- When documenting architecture decisions
- When onboarding new team members to the codebase
- When establishing or following contribution guidelines

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.
