# Global Validation

## When to use this skill:

- When validating user input from forms
- When validating API request bodies and parameters
- When validating query strings and URL parameters
- When implementing client-side validation for immediate feedback
- When implementing server-side validation (always required)
- When sanitizing input to prevent SQL injection, XSS, or command injection
- When validating data types, formats, and ranges
- When checking required fields and optional defaults
- When implementing business rule validation
- When writing validation error messages
- When using validation libraries (Zod, Yup, Joi, class-validator)
- When defining allowlists for acceptable input values
- When ensuring consistent validation across all entry points
- When reviewing code for validation coverage and security

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global validation.
