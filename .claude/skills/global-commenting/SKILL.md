# Global Commenting

## When to use this skill:

- When deciding whether to add a comment to code
- When writing comments to explain complex logic
- When documenting function or method purposes
- When adding JSDoc, docstrings, or similar documentation comments
- When reviewing existing comments for relevance and accuracy
- When removing outdated or misleading comments
- When ensuring code is self-documenting through clear naming
- When explaining the "why" behind non-obvious implementation decisions
- When writing comments that will remain relevant over time
- When avoiding comments about recent changes or temporary fixes
- When working on any code file that may benefit from explanation
- When refactoring code to be clearer instead of adding comments

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global commenting.
