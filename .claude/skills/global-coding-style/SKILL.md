# Global Coding Style

## When to use this skill:

- When writing any new code in the project
- When modifying existing code files
- When naming variables, functions, classes, methods, or files
- When deciding on function size and responsibility
- When organizing code within files and modules
- When applying consistent indentation and formatting
- When removing unused code, imports, or commented-out blocks
- When refactoring to reduce code duplication (DRY principle)
- When reviewing code for readability and maintainability
- When ensuring consistent style across different parts of the codebase
- When choosing descriptive, meaningful names over abbreviations
- When deciding whether to extract reusable functions
- When working on any .js, .ts, .jsx, .tsx, .py, .rb, .go, .java, or other code files

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global coding style.
