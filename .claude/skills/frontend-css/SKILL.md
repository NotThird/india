# Frontend CSS

## When to use this skill:

- When writing CSS, SCSS, SASS, LESS, or Stylus files
- When using Tailwind CSS utility classes
- When implementing CSS-in-JS (styled-components, Emotion, CSS modules)
- When working on files in `styles/`, `css/`, or similar directories
- When styling React, Vue, Svelte, or other framework components
- When implementing design tokens (colors, spacing, typography, shadows)
- When maintaining design system consistency
- When avoiding excessive framework style overrides
- When optimizing CSS for production (purging unused styles)
- When organizing CSS following BEM, SMACSS, or other methodologies
- When creating or modifying component-specific styles
- When reviewing CSS for consistency with project conventions
- When configuring Tailwind, PostCSS, or other CSS tooling

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend CSS.
