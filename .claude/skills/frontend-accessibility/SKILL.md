# Frontend Accessibility

## When to use this skill:

- When creating or modifying UI components
- When working on HTML, JSX, TSX, Vue, or Svelte template files
- When implementing interactive elements (buttons, links, forms, menus)
- When adding images and need to write alt text
- When building form inputs and labels
- When implementing keyboard navigation
- When managing focus in modals, dialogs, or dynamic content
- When choosing colors and ensuring sufficient contrast ratios
- When using or adding ARIA attributes
- When structuring heading hierarchies (h1-h6)
- When building navigation menus and skip links
- When creating dropdown menus, accordions, or tab interfaces
- When testing with screen readers
- When reviewing UI code for WCAG compliance
- When using semantic HTML elements (nav, main, article, button, etc.)

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend accessibility.
